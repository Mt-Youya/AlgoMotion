# Product

## Register

brand

## Users

AlgoMotion serves students, self-taught developers, interview candidates, and visually oriented engineers who want to understand an algorithm by running and manipulating it rather than reading static pseudocode. The homepage must work for a curious first-time visitor and signal enough technical depth to retain experienced developers.

## Product Purpose

AlgoMotion is a cross-platform algorithm learning and visualization environment. It turns trace events from real TypeScript and Rust/WASM implementations into interactive playback, synchronized code, complexity statistics, and exportable teaching videos. The homepage should demonstrate this behavior immediately and make the product feel tangible before asking visitors to read about it.

## Brand Personality

Kinetic, precise, and curious. The voice is concise and technically credible, with moments of controlled playfulness. It should feel like a beautifully engineered scientific toy rather than a course marketplace or a generic developer tool.

## Anti-references

- Generic centered SaaS heroes followed by interchangeable icon cards.
- Dark neon cyberpunk visuals used as shorthand for technology.
- Passive course-library pages that explain features without demonstrating them.
- Decorative motion that does not reveal algorithm state.
- Editorial magazine styling unrelated to an interactive learning product.

## Design Principles

1. Demonstrate before explaining.
2. Every motion should expose algorithm intent or state.
3. Let technical precision and playful interaction coexist.
4. Build one memorable interaction per chapter instead of adding decorative noise.
5. Keep the same visual language useful across web, desktop, mobile, and video.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Preserve complete keyboard access, visible focus treatment, semantic structure, sufficient color contrast, and a reduced-motion mode. Do not encode algorithm state by color alone; pair color with shape, position, labels, or text.
